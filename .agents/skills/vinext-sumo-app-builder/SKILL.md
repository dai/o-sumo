# vinext-sumo-app-builder

このスキルは、**Cloudflare vinext** フレームワークを活用して、大相撲の情報（番付、力士データ、星取表など）をモダンなUIで提供するWebアプリケーションを構築するための手順とリソースを提供します。

## 主な機能とワークフロー

このスキルを使用することで、以下のタスクを効率的に実行できます。

1. **プロジェクトの初期化と環境構築**
   - Cloudflare vinextの導入とViteベースのビルド設定
   - TypeScript、React、および大相撲アプリ向けのディレクトリ構造のセットアップ
2. **大相撲データの収集と統合**
   - 公式サイトやWikipedia、相撲レファレンスからの力士情報の抽出
   - 型定義に基づいた力士データ（`Rikishi`）と番付グループ（`RankGroup`）の作成
3. **番付・星取表ページのフロントエンド実装**
   - 東西の力士を並べた見やすい番付表（`BanzukeTable`）の構築
   - 幕内、十両などの各階級に応じたUIデザインの適用
4. **モダンなスタイリングとレスポンシブ対応**
   - 紫色のグラデーションを基調とした洗練されたデザインの適用
   - モバイルからデスクトップまで対応するレスポンシブレイアウトの実装
5. **Cloudflare Workersへのデプロイ**
   - `wrangler.toml` の設定と、`vinext deploy` を使用したデプロイ

## 開発のベストプラクティス

### データモデルの管理
力士データは `app/lib/sumo-data.ts` で一括管理し、型安全性を確保します。データの構造については、`references/sumo_data_schema.md` を参照してください。

### コンポーネントの設計
番付表示には、再利用可能な `BanzukeTable` コンポーネントを使用します。東西の力士を比較しやすく配置し、階級に応じたカラーバッジを適用することで視認性を高めます。

### スタイリング
プロジェクト全体で統一感のあるデザインを維持するため、`references/style_guide.md` に記載されたカラーパレットとレスポンシブブレークポイントを遵守してください。

## 便利なリソース

- **テンプレート**: `templates/app_starter.tsx` - 新しい場所の番付ページを作成する際のスターターコード
- **データ定義**: `references/sumo_data_schema.md` - 力士および番付データの型定義
- **デザインガイド**: `references/style_guide.md` - 配色、タイポグラフィ、レスポンシブ設定

## クイックスタート

新しい場所の番付ページ（例：令和8年3月場所）を追加する場合の最短手順です。

1. `app/lib/sumo-data.ts` に最新の力士データを追加します。
2. `app/[slug]/page.tsx` を作成し、`templates/app_starter.tsx` をベースに実装します。
3. `BanzukeTable` コンポーネントを使用して、各階級の番付を表示します。
4. `npm run dev` でローカル確認を行い、`npm run deploy` で公開します。
