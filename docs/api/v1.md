# o-sumo Public API v1

o-sumo は静的 JSON を配信します。現在の公開エンドポイントは以下です。

- `GET /api/v1/banzuke.json`
- `GET /api/v1/torikumi.json`

ベース URL 例: `https://osada.us`

## 共通仕様

- 形式: `application/json; charset=utf-8`
- 更新: GitHub Actions により定期更新（詳細は `docs/api/policy.md`）
- 文字列: 日本語を含む UTF-8
- 日付: `YYYY-MM-DD`（`updatedAt` / `isoDate`）

共通トップレベル項目:

- `bashoName`: 例 `"三月場所"`
- `year`: 例 `"令和八年"`
- `updatedAt`: 最終生成日（JST基準の日付文字列）

## `GET /api/v1/banzuke.json`

番付データです。

### レスポンス構造

- `makuuchi`: 幕内の段配列（例: 横綱, 大関, ...）
- `juryo`: 十両の段配列

段オブジェクト:

- `title`: 段名
- `east`: 東の力士配列
- `west`: 西の力士配列

力士オブジェクト:

- `id`: 力士ID（数値）
- `name`: しこ名
- `yomi`: よみ
- `rank`: 番付
- `side`: `east` or `west`
- `wins` / `losses` / `draws`: 星数
- `results`: 勝敗配列（`win` / `loss` / `draw`）
- `profileUrl`: 協会プロフィール URL
- `memo`: 備考

## `GET /api/v1/torikumi.json`

取組結果・予定データです。

### レスポンス構造

- `today`: 当日の結果/予定（`makuuchi`, `juryo`）
- `tomorrow`: 翌日の予定（`makuuchi`, `juryo`）
- `resultDays`: 15日分の結果アーカイブ
- `scheduleDays`: 15日分の予定アーカイブ
- `resultUpdatedAt`: 結果アーカイブの更新日
- `scheduleUpdatedAt`: 予定アーカイブの更新日

`today.makuuchi` などの division オブジェクト:

- `day`: 取組日番号（1-15）
- `dayName`: 例 `"取組日 十四日目"`
- `dayHead`: 例 `"十四日目： 令和8年3月21日(土)"`
- `division`: `"幕内"` / `"十両"`
- `matches`: 取組配列

`matches[]`:

- `division`
- `boutNo`
- `eastYomi` / `westYomi`
- `eastName` / `westName`
- `eastProfileUrl` / `westProfileUrl`
- `eastRank` / `westRank`
- `eastEnglish` / `westEnglish`
- `kimarite`
- `winner`（`east` / `west` / `null`）

`resultDays[]` / `scheduleDays[]` の日別オブジェクト:

- `day`, `isoDate`, `pathDate`, `label`, `dayHead`
- `status`: `published` or `pending`
- `statusMessage`: 結果なら `"結果未更新"`、予定なら `"取組予定未更新"`、公開済みなら `null`
- `data`: `{ makuuchi, juryo }`

### pending 日の扱い

未更新日の場合 `status = "pending"` になります。クライアント側では次を推奨します。

1. 対象日のページやカード自体は表示し、URLを固定化する
2. 取組表の代わりに `statusMessage` を表示する
3. 空配列を「エラー」とみなさない
4. `statusMessage` は結果なら `"結果未更新"`、予定なら `"取組予定未更新"` になる
5. `pending` 日の `matches` は空配列になる

### レスポンス例（抜粋）

## バージョニング

- 破壊的変更（必須項目の削除・型変更）は `v2` を新設
- `v1` では後方互換を維持した追加のみ実施
- 変更履歴は `docs/api/changelog.md` を参照
