# o-sumo Public API v1

[日本語版](./v1.md)

o-sumo serves static JSON. The currently published endpoints are:

- `GET /api/v1/banzuke.json`
- `GET /api/v1/torikumi.json`

Example base URL: `https://osada.us`

## Common Rules

- Content type: `application/json; charset=utf-8`
- Refresh model: periodically updated by GitHub Actions. See `docs/api/policy.en.md`
- Encoding: UTF-8, including Japanese strings
- Date format: `YYYY-MM-DD` for `updatedAt` and `isoDate`

Common top-level fields:

- `bashoName`: for example `"三月場所"`
- `year`: for example `"令和八年"`
- `updatedAt`: last generation date in JST

## `GET /api/v1/banzuke.json`

Banzuke data.

### Response Shape

- `makuuchi`: array of rank groups such as Yokozuna and Ozeki
- `juryo`: array of rank groups

Rank group object:

- `title`: rank title
- `east`: east-side rikishi array
- `west`: west-side rikishi array

Rikishi object:

- `id`: rikishi ID
- `name`: shikona
- `yomi`: reading
- `rank`: banzuke label
- `side`: `east` or `west`
- `wins` / `losses` / `draws`: record totals
- `results`: per-day marks using `win` / `loss` / `draw`
- `profileUrl`: Kyokai profile URL
- `memo`: notes

{
  "bashoName": "三月場所",

  "updatedAt": "2026-03-22",
  "makuuchi": [

          "losses": 4,

      ],

## `GET /api/v1/torikumi.json`

Torikumi results and schedule data.

### Response Shape

- `today`: current-day data with `makuuchi` and `juryo`
- `tomorrow`: next-day schedule data with `makuuchi` and `juryo`
- `resultDays`: 15-day results archive
- `scheduleDays`: 15-day schedule archive
- `resultUpdatedAt`: last results archive update date
- `scheduleUpdatedAt`: last schedule archive update date

Division object such as `today.makuuchi`:

- `day`: tournament day number, `1-15`
- `dayName`: for example `"取組日 十四日目"`
- `dayHead`: for example `"十四日目： 令和8年3月21日(土)"`
- `division`: `"幕内"` or `"十両"`
- `matches`: match array

`matches[]`:

- `division`
- `boutNo`
- `eastYomi` / `westYomi`
- `eastName` / `westName`
- `eastProfileUrl` / `westProfileUrl`
- `eastRank` / `westRank`
- `eastEnglish` / `westEnglish`
- `kimarite`
- `winner`: `east` / `west` / `null`

Daily archive object in `resultDays[]` or `scheduleDays[]`:

- `day`, `isoDate`, `pathDate`, `label`, `dayHead`
- `status`: `published` or `pending`
- `statusMessage`: `"結果未更新"` for results, `"取組予定未更新"` for schedules, or `null` when published
- `data`: `{ makuuchi, juryo }`

### Handling `pending` Days

When a day is not published yet, `status = "pending"`.

Recommended client behavior:

1. Keep the day page and URL available.
2. Show `statusMessage` instead of treating the day as missing.
3. Do not treat empty `matches` arrays as an error.
4. Expect `statusMessage` to be `"結果未更新"` for results or `"取組予定未更新"` for schedules.
5. Expect `matches` to be empty arrays on pending days.

  "year": "令和八年",
  "updatedAt": "2026-03-22",

    "makuuchi": {
      "day": 14,
      "dayName": "取組日 十四日目",
      "dayHead": "十四日目： 令和8年3月21日(土)",
      "division": "幕内",
      "matches": [
        {
          "division": "幕内",
          "boutNo": 1,
          "eastName": "あさこうりゅう",
          "eastYomi": "あさこうりゅう",
          "eastEnglish": "Asakoryu",
          "eastRank": "前頭十二枚目",
          "eastProfileUrl": "https://www.sumo.or.jp/ResultRikishiData/profile/4101/",

List of basic rikishi information.

### Response Shape

- `updatedAt`: last generation date in JST
- `rikishi`: array of rikishi

Rikishi basic info object:

- `id`: rikishi ID
- `name`: shikona
- `yomi`: reading
- `currentRank`: current banzuke
- `profileUrl`: Kyokai profile URL

### Response Example

```json
{
  "updatedAt": "2026-03-30",
  "rikishi": [
    {
      "id": 3842,
      "name": "豊昇龍",
      "yomi": "ほうしょうりゅう",
      "currentRank": "横綱",
      "profileUrl": "https://www.sumo.or.jp/ResultRikishiData/profile/3842/"
    }
  ]
}
```

- `v1` only accepts backward-compatible additions
