# o-sumo Public API v1

[日本語版](./v1.md)

o-sumo serves static JSON. The currently published endpoints are:

- `GET /api/v1/banzuke.json`
- `GET /api/v1/torikumi.json`
- `GET /api/v1/rikishi.json`
- `GET /api/v1/rikishi/{id}.json`

Example base URL: `https://osada.us`

## Common Rules

- Content type: `application/json; charset=utf-8`
- Refresh model: periodically updated by GitHub Actions. See `docs/api/policy.en.md`
- Encoding: UTF-8, including Japanese strings
- Date format: `YYYY-MM-DD` for `updatedAt` and `isoDate`

Common top-level fields for banzuke/torikumi endpoints:

- `bashoName`: for example `"五月場所"`
- `year`: for example `"令和八年"`
- `updatedAt`: last generation date in JST

After the May 2026 banzuke release on April 27, 2026, `banzuke.json` switches to the May basho banzuke. `torikumi.json` uses the same `202605` month key and keeps unpublished torikumi days as `pending`.

## `GET /api/v1/banzuke.json`

Banzuke data.

### Response Shape

- `makuuchi`: array of rank groups such as Yokozuna and Ozeki
- `juryo`: array of rank groups

Rank group object:

- `title`: rank title
- `east`: east-side rikishi array
- `west`: west-side rikishi array

Rikishi object:

- `id`: rikishi ID
- `name`: shikona
- `yomi`: reading
- `rank`: banzuke label
- `side`: `east` or `west`
- `wins` / `losses` / `draws`: record totals
- `results`: per-day marks using `win` / `loss` / `draw`
- `profileUrl`: Kyokai profile URL
- `memo`: notes

### Response Example

```json
{
  "bashoName": "五月場所",
  "year": "令和八年",
  "updatedAt": "2026-04-14",
  "makuuchi": [
    {
      "title": "横綱",
      "east": [
        {
          "id": 3842,
          "name": "豊昇龍",
          "yomi": "ほうしょうりゅう",
          "rank": "横綱",
          "side": "east",
          "wins": 11,
          "losses": 4,
          "draws": 0,
          "results": ["win", "win", "win", "loss"],
          "profileUrl": "https://www.sumo.or.jp/ResultRikishiData/profile/3842/",
          "memo": ""
        }
      ],
      "west": [
        {
          "id": 4227,
          "name": "大の里",
          "yomi": "おおのさと",
          "rank": "横綱",
          "side": "west",
          "wins": 0,
          "losses": 4,
          "draws": 11,
          "results": ["loss", "loss", "loss", "loss"],
          "profileUrl": "https://www.sumo.or.jp/ResultRikishiData/profile/4227/",
          "memo": ""
        }
      ]
    }
  ]
}
```

## `GET /api/v1/torikumi.json`

Torikumi results and schedule data.

### Response Shape

- `today`: current-day data with `makuuchi` and `juryo`
- `tomorrow`: next-day schedule data with `makuuchi` and `juryo`
- `resultDays`: 15-day results archive
- `scheduleDays`: 15-day schedule archive
- `resultUpdatedAt`: last results archive update date
- `scheduleUpdatedAt`: last schedule archive update date

Division object such as `today.makuuchi`:

- `day`: tournament day number, `1-15`
- `dayName`: for example `"取組日 十四日目"`
- `dayHead`: for example `"初日： 令和8年5月10日(日)"`
- `division`: `"幕内"` or `"十両"`
- `matches`: match array

`matches[]`:

- `division`
- `boutNo`
- `eastYomi` / `westYomi`
- `eastName` / `westName`
- `eastProfileUrl` / `westProfileUrl`
- `eastRank` / `westRank`
- `eastEnglish` / `westEnglish`
- `kimarite`
- `winner`: `east` / `west` / `null`

Daily archive object in `resultDays[]` or `scheduleDays[]`:

- `day`, `isoDate`, `pathDate`, `label`, `dayHead`
- `status`: `published` or `pending`
- `statusMessage`: `"結果未更新"` for results, `"取組予定未更新"` for schedules, or `null` when published
- `data`: `{ makuuchi, juryo }`

### Handling `pending` Days

When a day is not published yet, `status = "pending"`.

Recommended client behavior:

1. Keep the day page and URL available.
2. Show `statusMessage` instead of treating the day as missing.
3. Do not treat empty `matches` arrays as an error.
4. Expect `statusMessage` to be `"結果未更新"` for results or `"取組予定未更新"` for schedules.
5. Expect `matches` to be empty arrays on pending days.

### Response Example

```json
{
  "bashoName": "五月場所",
  "year": "令和八年",
  "updatedAt": "2026-04-14",
  "resultUpdatedAt": "2026-04-14",
  "scheduleUpdatedAt": "2026-04-14",
  "resultDays": [
    {
      "day": 1,
      "isoDate": "2026-05-10",
      "pathDate": "20260510",
      "label": "初日",
      "dayHead": "初日： 令和8年5月10日(日)",
      "status": "pending",
      "statusMessage": "結果未更新",
      "data": {
        "makuuchi": {
          "day": 1,
          "dayName": "取組日 初日",
          "dayHead": "初日： 令和8年5月10日(日)",
          "division": "幕内",
          "matches": []
        },
        "juryo": {
          "day": 1,
          "dayName": "取組日 初日",
          "dayHead": "初日： 令和8年5月10日(日)",
          "division": "十両",
          "matches": []
        }
      }
    }
  ]
}
```

On actual `pending` days, the daily archive object keeps the same shape, but `status` becomes `pending`, `statusMessage` becomes `"結果未更新"` or `"取組予定未更新"`, and `matches` stays empty.

## `GET /api/v1/rikishi.json`

List of basic rikishi information.

### Response Shape

- `updatedAt`: last generation date in JST
- `rikishi`: array of rikishi

Rikishi basic info object:

- `id`: rikishi ID
- `name`: shikona
- `yomi`: reading
- `currentRank`: current banzuke
- `profileUrl`: Kyokai profile URL

### Response Example

```json
{
  "updatedAt": "2026-04-14",
  "rikishi": [
    {
      "id": 3842,
      "name": "豊昇龍",
      "yomi": "ほうしょうりゅう",
      "currentRank": "横綱",
      "profileUrl": "https://www.sumo.or.jp/ResultRikishiData/profile/3842/"
    }
  ]
}
```

## `GET /api/v1/rikishi/{id}.json`

Detailed profile for a specific rikishi ID. Use an `id` value from `rikishi.json`.

### Response Shape

- `id`: rikishi ID
- `name`: shikona
- `yomi`: reading
- `currentRank`: current banzuke rank
- `birthDate`: birth date text (empty string if unavailable)
- `height`: height in cm
- `weight`: weight in kg
- `shusshin`: hometown or profile-origin text
- `debut`: debut basho text
- `careerStats`: career totals object
  - `wins`: total wins
  - `losses`: total losses
  - `draws`: total absences
- `photoUrl`: profile image URL (typically `/images/rikishi/{id}.png`). o-sumo prefers a local image processed with MiniMax I2I Generation from a Japan Sumo Association profile photo, or an empty string if unavailable
- `sourceUrl`: Kyokai profile URL
- `updatedAt`: last generation date for the detail JSON in JST

### Response Example

```json
{
  "id": 3842,
  "name": "豊昇龍",
  "yomi": "ほうしょうりゅう",
  "currentRank": "横綱",
  "birthDate": "平成11年5月22日（26歳）",
  "height": 188,
  "weight": 148,
  "shusshin": "東大関",
  "debut": "平成三十年一月場所",
  "careerStats": {
    "wins": 0,
    "losses": 0,
    "draws": 0
  },
  "photoUrl": "/images/rikishi/3842.png",
  "sourceUrl": "https://www.sumo.or.jp/ResultRikishiData/profile/3842/",
  "updatedAt": "2026-04-27"
}
```

## Versioning

- Breaking changes such as removing required fields or changing field types should be published as `v2`
- `v1` only accepts backward-compatible additions
- See `docs/api/changelog.en.md` for change history
